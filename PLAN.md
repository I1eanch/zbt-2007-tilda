# План корректировки маркетинговых формулировок

## Context

Нужно привести тексты лендинга марафона к более осторожным формулировкам: убрать конкретные суммы заработка, гарантированный результат и категоричные обещания о здоровье, понимании состояния и эффекте программы. Цель — сохранить смысл и привлекательность текста, но описывать возможности, подходы и первые шаги без обещания результата.

## Approach

Обновить найденные тексты в двух слоях страницы: Astro-источниках (`Hero.astro`, `DualValue.astro`, `landing.ts`) и standalone HTML для Tilda. Для каждой фразы использовать один согласованный нейтральный вариант, затем синхронизировать основной и preview HTML. Не затрагивать смысловые блоки и неизменяемый референс `Maket.html`; проверить, что исходные обещающие варианты и диапазон дохода больше не попадают в пользовательский интерфейс.

## Files to modify

- `src/components/Hero.astro` — вводный текст о трёх днях и профессии без диапазона дохода.
- `src/components/DualValue.astro` — заголовок о первых шагах к восстановлению, описание применения рекомендаций во время марафона, карьерное описание и proof-карточка без точного заработка.
- `src/content/landing.ts` — хук первого дня и пункт программы о работе специалиста над собственными привычками.
- `tilda-zbt.html` — все согласованные смысловые замены в standalone-версии страницы, включая три вхождения диапазона дохода.
- `tilda-zbt.preview.html` — синхронная копия новых формулировок для локального preview.

## Reuse

- Текущую архитектуру хранения контента: статические тексты секций остаются в соответствующих Astro-компонентах, данные программы — в `src/content/landing.ts`.
- Существующий `npm run verify`, включающий проектные проверки, и Playwright-тест `tests/tilda-block.spec.ts` — использовать для итоговой проверки.
- Ручную синхронизацию standalone-разметки между `tilda-zbt.html` и `tilda-zbt.preview.html`: отдельного генератора HTML в проекте нет; `scripts/build-tilda-assets.mjs` оптимизирует только изображения. `Maket.html` не редактировать — по `AGENTS.md` это неизменяемый референс.

## Steps

- [ ] В `Hero.astro` заменить подзаголовок на: «Три дня на пути к пониманию своего состояния и тому, как знания о здоровье могут стать основой профессии с возможностью достойного заработка из любой точки мира».
- [ ] В `DualValue.astro` заменить заголовок на «Сделаете первые шаги к восстановлению здоровья — и узнаете, как на этом зарабатывают», а финал карточки здоровья — на «Получите рекомендации, которые сможете начать применять уже во время марафона».
- [ ] В карьерной карточке `DualValue.astro` заменить диапазон на качественное описание «Узнаете о возможностях достойного заработка в нутрициологии», а числовую proof-карточку `$800–5 000 / в месяц зарабатывают…` — на нейтральные `Достойный заработок / одна из возможностей развития в профессии`. Рыночный показатель `$400+ млрд` не менять: это не сумма обещанного заработка.
- [ ] В `landing.ts` заменить финал хука первого дня на: «За вечер вы узнаете об одном из подходов к поиску возможных причин нарушений самочувствия и по-новому взглянете на здоровье».
- [ ] В `landing.ts` заменить пункт о «продукте своего продукта» на: «Для многих знакомство с нутрициологией начинается с работы над собственными привычками и образом жизни».
- [ ] Вручную перенести те же формулировки в `tilda-zbt.html` и `tilda-zbt.preview.html`, сохранив две HTML-копии текстово синхронными в затронутых местах.
- [ ] Проверить отсутствие всех старых формулировок и точного диапазона заработка, затем выполнить проектную верификацию и визуальную проверку.

## Verification

- Выполнить статический поиск по полным старым фразам, `$800`, `$5 000` и вариантам записи диапазона во всех Astro/Tilda-источниках; совпадений в пользовательском контенте быть не должно.
- Запустить `npm run verify`: общий gate последовательно проверит форму контента, Astro-типы, production-сборку и весь Playwright-набор, включая `tests/tilda-block.spec.ts` для standalone `tilda-zbt.html`.
- Вручную проверить Astro-страницу и локальный `tilda-zbt.preview.html` на desktop/mobile, особенно переносы строк в hero, заголовке секции и длинных карточках программы. Container smoke не относится к standalone Tilda-тексту и для этой правки не требуется.

## Decisions

- Использовать перечисленные в шагах консервативные варианты из рекомендаций маркетолога как единый текст для всех реализаций страницы.
- Обновить обе отслеживаемые HTML-копии вручную: генератора Tilda HTML в репозитории нет.
- Не изменять `$400+ млрд`, поскольку это оценка размера рынка, а не обещание личного заработка.
- Не менять тестовые файлы: существующие проверки не зависят от старых фраз; статический поиск подтверждает полноту текущих замен, а `npm run verify` проверяет сборку и отсутствие функциональных/визуальных регрессий.
