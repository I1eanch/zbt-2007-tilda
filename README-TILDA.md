# Здоровье без таблеток — лендинг для Tilda (блок T123)

Готовый **самодостаточный** HTML-блок лендинга для вставки в Tilda одним блоком
«HTML-код». Ни iframe, ни внешнего хостинга: весь дизайн, вёрстка и логика — в
одном файле `tilda-zbt.html`.

## Что в комплекте

| Файл | Назначение |
|---|---|
| `tilda-zbt.html` | **Код для вставки в Tilda** — весь лендинг одним блоком |
| `README-TILDA.md` | Эта инструкция |

Изображения загружать **не нужно** — все иллюстрации сделаны инлайн-SVG прямо в
коде (легко, чётко на любом экране, ничего не «отвалится»).

## Установка в Tilda (3 шага)

**1. Добавьте HTML-блок.**
На нужной странице: «Добавить блок» → раздел **«Другое» → «T123 · HTML-код»**.
Ставьте обычным блоком в потоке страницы (не внутри Zero Block/артборда).

**2. Вставьте код.**
Откройте `tilda-zbt.html`, скопируйте **всё содержимое файла** и вставьте в поле
кода блока T123. Сохраните, опубликуйте.

**3. Проверьте регистрацию.**
Все кнопки «Забрать место» плавно прокручивают к блоку регистрации, где
встроена ваша форма-виджет GetCourse (`iandmyhealth.ru`, id `1629532`). Форма
грузится автоматически при приближении к блоку. Заявки идут в GetCourse, как
сейчас на сайте.

Готово. Больше ничего настраивать не нужно.

## Как это устроено

- **Изоляция.** Весь CSS вложен в область `.zbt`, а классы имеют префикс
  `.zbt-…`. Стили лендинга **не протекают** в шапку/подвал Tilda, а стили Tilda
  не ломают лендинг.
- **JS.** Обёрнут в IIFE с защитой `data-zbt-init` на корневом элементе — при
  ре-рендере блока в редакторе Tilda обработчики не дублируются.
- **Шрифты.** Inter + Montserrat подключаются и через `<link>`, и через `@import`
  внутри `<style>` (Tilda иногда вырезает верхний `<link>`).
- **Регистрация.** Форма-виджет GetCourse монтируется лениво (при подходе блока
  `#zbt-reg` к экрану или по клику на любой CTA) — до этого внешняя сеть не
  задействуется. Виджет сам отправляет заявки на свою платформу.
- **Плавающая кнопка.** Появляется после первого экрана и прячется, когда виден
  блок регистрации.
- **Навигация.** Фиксированной шапки-меню в блоке нет — она намеренно опущена
  (как в прототипе `Maket.html` и в референсе `Site_ZBT_RU`). Внутри блока T123
  фиксированное меню конфликтовало бы с собственной навигацией страницы Tilda и
  было бы ненадёжным. Навигацию/меню обеспечивает сама страница Tilda; лендинг
  идёт единым потоком с плавающей кнопкой-CTA.
- **Анимации.** Только `transform`/`opacity`, есть ветка `prefers-reduced-motion`.
- **Адаптив.** Проверено на 1440 / 768 / 375 px, без горизонтального скролла.

## Настройка

- **Другая форма регистрации.** В `<script>` в конце файла замените константы
  `WIDGET_ID` и `WIDGET_SRC` на значения вашего виджета GetCourse
  («Сайт → Формы → Код для вставки»).
- **Тексты/даты.** Правятся прямо в разметке внутри `.zbt` (даты «20–22 июля»,
  цифры, подарки).

## Если что-то не так

| Симптом | Причина / решение |
|---|---|
| Форма не появляется | Проверьте `WIDGET_ID`/`WIDGET_SRC`; проверьте, что виджет активен в GetCourse |
| Шрифты не подгрузились | Tilda обрезал `<link>` — `@import` в начале `<style>` уже дублирует их; проверьте, что вставлен **весь** блок |
| Поехала вёрстка Tilda | Не должно быть — стили scoped под `.zbt`. Убедитесь, что вставлен весь блок целиком, включая `<style>` |
| Плавающая кнопка перекрывает контент | Это ожидаемо только на мобильных внизу; она прячется на блоке регистрации |

## Связь с Astro-версией сайта

В репозитории есть полноценная Astro-версия сайта (каталог `src/`, сборка в
`dist/`, контейнер Docker/nginx). Она предназначена для **самостоятельного**
хостинга и по умолчанию запрещает встраивание во фрейм
(`X-Frame-Options: DENY`) — то есть напрямую в iframe Tilda не вставляется.

Этот файл (`tilda-zbt.html`) — **выбранный способ размещения в Tilda**: один
нативный самодостаточный блок, без внешнего хостинга и без iframe, по образцу
проекта `Site_ZBT_RU`.
