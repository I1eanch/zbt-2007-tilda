# Здоровье без таблеток — лендинг для Tilda (блок T123)

Лендинг одним блоком для вставки в Tilda. Весь дизайн, вёрстка и логика — в файле
`tilda-zbt.html`. Фотографии хранятся в этом же GitHub-репозитории (папка
`tilda-assets/`) и отдаются по прямым ссылкам через CDN **jsDelivr**. Ничего
загружать в Tilda и ничего заменять в коде не нужно.

## Установка (2 шага)

**1. Запушьте репозиторий на GitHub** (он должен быть **публичным** — иначе
jsDelivr не отдаст фото). Фото уже лежат в `tilda-assets/` и поедут вместе с
репозиторием. Ссылки в коде уже указывают на них:
`https://cdn.jsdelivr.net/gh/I1eanch/zbt-2007-tilda@main/tilda-assets/…`

**2. Вставьте код в Tilda.**
На странице: «Добавить блок» → **«Другое» → «T123 · HTML-код»** (обычным блоком в
потоке, не внутри Zero Block). Скопируйте **всё** содержимое `tilda-zbt.html`,
вставьте, сохраните, опубликуйте.

Готово — фото подтянутся автоматически.

> Посмотреть, как выглядит блок локально (без публикации): откройте
> `tilda-zbt.preview.html` в браузере — там ссылки временно заменены на локальные
> файлы из `tilda-assets/`.

## Важные детали

- **Репозиторий должен быть публичным.** jsDelivr (и `raw.githubusercontent.com`)
  отдают файлы только из публичных репозиториев.
- **Имя репозитория / ветка.** Ссылки указывают на
  `I1eanch/zbt-2007-tilda@main`. Если репозиторий назван иначе или ветка не
  `main` — поправьте базовый URL в `tilda-zbt.html` (найдите
  `cdn.jsdelivr.net/gh/` и замените `I1eanch/zbt-2007-tilda@main` на своё).
- **Кэш jsDelivr.** По ветке `@main` CDN кэширует файлы ~7 дней. Если заменили
  фото и хотите увидеть сразу — используйте в URL конкретный тег/коммит вместо
  `@main`, либо запросите очистку через `https://purge.jsdelivr.net/gh/…`.
- **Обновить/пережать фото:** положите исходники в `src/assets/` и выполните
  `node scripts/build-tilda-assets.mjs` — оптимизированные файлы обновятся в
  `tilda-assets/`.

## Как это устроено

- **Изоляция.** Весь CSS вложен в `.zbt`, классы с префиксом `.zbt-…`, значения —
  из дизайн-токенов проекта (`tokens.css`). Стили лендинга не текут в Tilda и
  наоборот (проверяется тестом `tests/tilda-block.spec.ts`).
- **Шапка-меню** (`Программа/Профессия/Спикер/Бонусы` + «Забрать место») —
  `sticky` вверху блока, с мобильным бургер-меню; пункты плавно скроллят к
  секциям (с отступом под высоту шапки). В блоке T123 намеренно `sticky`, а не
  `fixed`, чтобы шапка не перекрывала другие блоки страницы Tilda.
- **Регистрация** — встроенный виджет-форма GetCourse (`iandmyhealth.ru`,
  id `1629532`), грузится лениво; заявки идут в GetCourse. Все кнопки «Забрать
  место» плавно скроллят к форме.
- **Плавающая кнопка** — появляется после первого экрана, прячется на hero и на
  блоке регистрации; скрытая недоступна с клавиатуры.
- **JS** — в IIFE с очисткой (`AbortController`) при ре-рендере блока Tilda.
- **Шрифты** — Inter + Montserrat (через `<link>` и дублирующий `@import`).
- **Адаптив** — проверено на 1440 / 768 / 375 px, без горизонтального скролла.
  (Мобильная раскладка опирается на `<meta viewport>` страницы Tilda — он есть по
  умолчанию.)

## Настройка контента

- **Другая форма регистрации:** в `<script>` внизу файла замените `WIDGET_ID` и
  `WIDGET_SRC`.
- **Тексты/даты** правятся прямо в разметке внутри `.zbt`.

## Связь с Astro-версией сайта

В репозитории есть полноценная Astro-версия сайта (`src/`, сборка в `dist/`,
контейнер Docker/nginx) для самостоятельного хостинга. Она по умолчанию запрещает
встраивание во фрейм (`X-Frame-Options: DENY`). Файл `tilda-zbt.html` — способ
разместить лендинг **прямо в Tilda** одним блоком, без внешнего хостинга и iframe.

## Если что-то не так

| Симптом | Причина / решение |
|---|---|
| Битые картинки | Репозиторий приватный, или имя репо/ветка в URL не совпадают, или ещё не запушено |
| Старое фото после замены | Кэш jsDelivr по `@main` (~7 дней) — используйте тег/коммит или purge.jsdelivr.net |
| Форма не появляется | Проверьте `WIDGET_ID`/`WIDGET_SRC` и что виджет активен в GetCourse |
| Шрифты не подгрузились | Вставлен не весь блок — проверьте, что скопирован `<style>` целиком |
